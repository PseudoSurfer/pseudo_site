Adding a hype banner

01 Apr 2021 - Pseudo Surfer

I’ve changed the home page by adding an animated banner.

I used Tumult Hype to create a HTML animation for my home page banner. It’s pretty straightforward figuring out how to use Hype if you’ve used After Effects or other animation type software in the past.

I used Blender to create my rolling text. I could’ve used Blender for a lot more of the banner but since I’m trying to learn how to use Hype, I decided to do just the main text piece. Blender saves animations as movie files or individual PNGs which is the preferred method to use in case Blender crashes or in case you need to pause your rendering you can just pick up where you left off.

Hype lets you add a sequence of images as a sprite sheet. Hype exports the final piece as an index.html file along with all the supporting pieces in a folder.

Adding your final piece to your Jekyll set-up is a little more difficult. Hype uses vanilla javascript and it outputs a minified version of the javascript file which helps it to load faster. However all the paths to the images, sounds, svg, and javascript files are written inside the minified javascript file which requires you to go in and add your Jekyll required path like :”/assets/images” to the paths of each file. I think I had a total of six which need to be changed.
